# restaurant-catalog-template

Универсальный шаблон электронного каталога ресторана на `React + TypeScript + Vite + Supabase`.

## Что уже подготовлено

- базовая архитектура `app / pages / widgets / features / entities / shared`
- безопасная инициализация приложения
- `ErrorBoundary`, `LoadingState`, `ErrorState`, `EmptyState`
- `HashRouter` для простого MVP на GitHub Pages
- базовый guard для `/admin/*`
- клиент Supabase и валидация env
- Zustand store для корзины
- SQL-миграции, seed-файлы и GitHub Actions workflow
- PWA-конфигурация и SVG-иконки-заглушки

## Быстрый старт

1. Установите Node.js 20+.
2. Скопируйте `.env.example` в `.env.local`.
3. Заполните `VITE_SUPABASE_URL` и `VITE_SUPABASE_PUBLISHABLE_KEY`.
4. Установите зависимости: `npm install`
5. Запустите проект: `npm run dev`

## Проверка перед публикацией

```bash
npm run check
```

## GitHub Pages

Для публикации на GitHub Pages не нужно загружать исходники как есть в статический хостинг.
В репозитории уже подготовлен workflow, который:

- устанавливает зависимости;
- собирает Vite-приложение;
- публикует именно папку `dist`.

Это важно, потому что при правильной публикации fallback-экран из `index.html` не останется на сайте: React-приложение загрузится и заменит его.

Если Supabase ещё не настроен, публичный каталог всё равно открывается в безопасном пустом режиме.

## Важное замечание

В текущем коммите собран фундамент шаблона и структура под MVP. Полный CRUD для ресторана, категорий, блюд, вариантов, дополнений, бронирований и интеграция с реальными данными Supabase остаются следующим этапом.
